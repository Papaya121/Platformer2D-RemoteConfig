# Platformer2D-RemoteConfig

## Описание

В проекте реализована система удалённой конфигурации оружия (`RemoteConfigLoader`), которая:

* загружает конфиг по URL (CSV)
* парсит данные
* валидирует поля
* сохраняет локальную копию в `Application.persistentDataPath`
* использует кэш при отсутствии сети
* использует дефолтные значения при отсутствии сети и кэша

Конфиг применяется к классу `Weapon`, который используется в `ShootingController`.

---

## Формат удалённого конфига

CSV файл со следующей структурой:

```
id,damage,cooldown
pistol,20,0.1
rifle,10,0.2
```

---

## Логика работы

### 1. При старте сцены:

* Применяются кэшированные значения, если их нет, то дефолтные.
* Запускается загрузка удалённого конфига.

### 2. Если загрузка успешна:

* CSV парсится.
* Выполняется валидация.
* Конфиг сохраняется в кэш.
* Значения применяются к оружию.

### 3. Если загрузка не удалась:

* Выполняется попытка загрузить локальный кэш.
* Если кэш валиден – применяется он.

### 4. Если кэш отсутствует или некорректен:

* Используются заранее заданные дефолтные значения.
* В консоль выводится сообщение об ошибке.

---

## Кэширование

Кэш сохраняется в:

```
Application.persistentDataPath/weapon_config_cache.csv
```

Путь можно увидеть в логах Unity при успешном сохранении кэша.
